PCAP Analyzer is the culmination of our twelve week summer internship with Google. Our project focused on designing a tool to help everyday computer users understand the basic fundamentals of how data is shared across the Internet.

Getting Started

Our project analyzed packets which are stored in PCAP (packet capture) files. By analyzing PCAP file you can tell a lot about your internet footprint. You can learn more about internet fundamentals below or view PCAP Visualization by click on the Start button in the top right-hand corner

How does the Internet Work?

When you load something on the internet, its content is split into tiny fragments called packets. These packets are sent over WIFI and network cabling via different types of protocols. What protocols is used determines how the packet is to be transported and interpreted by the receiving device. Once the packet is routed from one device to another, they packets are rebuild into the content originally requested.

Some Common protocols:
  • TCP: Core internet protocol with build in retransmission of lost packets
  • UDP: A faster core protocol then TCP without the lost packet ditection
  • DNS: The phonebook of the Internet, it converts IP address into domain names (TCP/UDP)
  • HTTP/S: protocol used to transmit Webpages (TCP/UDP)
  • IMAP/POP: protocol used to transmit Email (TCP)
